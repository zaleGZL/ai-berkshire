# 任意 Symbol 技术分析

对 `$ARGUMENTS` 中的 symbol 做技术分析。默认使用日线 2 年 + 周线 5 年；用户指定周期时以用户要求为准。

支持代码、公司名或资产名，例如 `AAPL`、`0700.HK`、`600519.SS`、`BTC-USD`、`EURUSD=X`、`GC=F`、`^GSPC`。

## 执行流程

### 1. 确认日期与标的

先运行 `date`，把当天日期作为数据时效基线。

解析并确认：标准 symbol、资产名称、交易所、资产类型、计价货币、交易时区。代码存在歧义时先搜索交易所或资产名；仍无法唯一确定时请用户确认，禁止猜测。

常见 Yahoo symbol 映射：

| 类型 | 示例 |
|------|------|
| 美股 / ETF | `AAPL` / `SPY` |
| 港股 | `0700.HK` |
| A 股 | `600519.SS` / `000001.SZ` |
| 指数 | `^GSPC` / `^HSI` |
| 加密货币 | `BTC-USD` |
| 外汇 | `EURUSD=X` |
| 期货 | `GC=F` |

### 2. 获取并计算数据

从仓库根目录运行共享工具。它复用现有 Yahoo Chart API 路径，以 Yahoo `Adj Close / Close` 比例同步调整 OHLC、按交易所时区剔除未完成 K 线，并以固定口径计算指标：

```bash
python3 tools/technical_analysis.py {SYMBOL} --range 2y --interval 1d
python3 tools/technical_analysis.py {SYMBOL} --range 5y --interval 1wk
```

只分析用户指定周期时，只运行对应数据窗口。工具输出的 `raw_close` 用于跨来源核对，`adjusted_close` 用于指标；禁止拿调整后历史价格与未复权页面直接比较。周线/月线的 `cutoff` 是周期起始标签，展示时写成“YYYY-MM-DD 当周/当月”，不要冒充周期结束日。

若 Yahoo 无该 symbol 或数据落后，先确认供应商代码映射，再改用能提供完整历史 OHLCV 的公开来源，并用 Python 按相同口径重算；不要凭图形截图估算指标。

### 3. 交叉验证

必须使用真正独立于 Yahoo 的第二来源。Yahoo 网页、Yahoo 搜索摘要或转述 Yahoo 的页面不算独立来源。

| 资产 | 优先核验来源 | 口径要求 |
|------|-------------|---------|
| 股票 / ETF | 上市交易所历史行情；其次独立数据商 | 同交易所、币种、交易日、原始收盘价 |
| 指数 | 指数编制方或官方交易所 | 现货指数，不与 ETF/期货替代品混比 |
| 期货 | 合约交易所结算价 | 同一合约月份；连续合约必须单独标记 |
| 加密货币 | 具名现货交易所或独立聚合源 | 同一交易对、同一 UTC 日界线 |
| 外汇 | 独立现货报价源 | 同一货币对和日界线；承认无集中收盘价 |

至少核对：

- 标的身份、交易所、币种
- 双方最新一根已完成 K 线的日期和原始收盘价
- 是否发生拆股、合约换月、币种转换或复权口径差异

执行两道门槛：

1. **新鲜度门槛**：第二来源已有更新的已完成 K 线时，主数据源已过期。先重取；仍落后则换完整 OHLCV 来源重算。无法重算时只报告“数据截止日状态”，把当前结论标为降级，禁止写成最新判断。
2. **一致性门槛**：仅比较同一日期、同一口径。使用 `python3 tools/financial_rigor.py cross-validate --field close --values '{"Yahoo": 数值, "来源2": 数值}' --unit {币种} --tolerance 1` 精确计算。差异超过 1% 时先排查实时价/收盘价、复权、主上市/ADR、连续合约/具体合约；仍无法解释则停止方向结论。

成交量只有在交易场所、单位和覆盖范围一致时才能跨源比较。否则只使用同一来源内的“当前量 / 20 期均量”，并标注其覆盖范围。

### 4. 分析顺序

按以下顺序解释，禁止用单一指标直接给买卖结论：

1. **价格结构**：1/5/20/60/252 根 K 线涨跌、52 周位置、20/60 周期高低区间
2. **趋势**：价格与 SMA20/50/200 的关系、均线排列、日线与周线是否同向
3. **动量**：RSI14、MACD(12,26,9) 及柱体方向；超买不等于立刻下跌，超卖不等于立刻反弹
4. **波动**：ATR14、ATR 占价格比例、布林带(20,2)，用于描述风险和波动扩张/收缩
5. **量价**：最新成交量相对 20 期均量；无可靠成交量的外汇、指数等标记为不可用
6. **关键位**：只使用工具输出的 20/60 周期高低点与主要均线，不编造精确支撑阻力
7. **情景**：给出偏多、震荡、偏空三个条件，以及使当前判断失效的价格或结构

### 5. 输出格式

默认先给数据可信度，再给分析；同一数值不要在“关键指标”和“关键位置”重复罗列。价格按标的最小变动单位或合理精度展示，百分比保留 2 位，禁止照抄工具的 4 位小数制造虚假精度。

```markdown
# {资产名称}（{SYMBOL}）技术分析

**分析日期**：YYYY-MM-DD
**数据截止**：日线 YYYY-MM-DD；周线 YYYY-MM-DD 当周
**市场 / 币种 / 周期**：...
**数据可靠性**：通过 / 降级 / 阻断

## 数据源核验
| 检查项 | 主来源 | 独立来源 | 差异 | 结果 |
|--------|-------|---------|------|------|
| 标的/市场/币种 | | | — | |
| 最新已完成日线 | 日期 + raw close | 日期 + close | x.xx% | |
| 复权/合约/日界线 | | | — | |

[主来源链接] · [独立来源链接]

## 一句话结论
[趋势状态 + 动量状态 + 最重要的确认/失效条件]

## 多周期仪表盘
| 周期 | 趋势 | 动量 | 波动 | 量价 | 结论 |
|------|------|------|------|------|------|

## 关键指标与价格结构
| 指标 | 数值 | 解读 |
|------|------|------|

## 关键位置与情景
| 情景 | 触发条件 | 失效条件 | 观察重点 |
|------|---------|---------|---------|

## 数据口径与局限
[Yahoo 调整因子、数据新鲜度、缺失成交量、历史长度、来源差异]
```

若数据可靠性为“阻断”，输出到“数据源核验”和“数据口径与局限”为止，不生成方向性结论。其他情况下，结论只能描述技术状态和条件，不把技术指标包装成确定性预测。技术分析不替代基本面研究，也不构成投资建议。

用户明确要求保存报告时，写入 `reports/{标的名称}/{标的名称}-technical-{YYYYMMDD}.md`，再运行 `python3 tools/report_audit.py {报告路径}`；未要求时直接在对话中返回，不创建文件。
